## 📄 하위 페이지: 클라우드 이용 모델
### 클라우드 이용 모델

  - 클라우드 컴퓨팅 서비스를 사용하는 방식
  - 각 모델은 제공되는 서비스 범위와 사용자의 관리 책임 범위가 다름
  - 대표적인 이용 모델은 제공하는 방식에 따라 IaaS, PaaS, SaaS 등이 있음
### as a Service

  - 클라우드 컴퓨팅 환경에서 다양한 IT 자원과 서비스를 필요에 따라 제공하는 방식
  - 주요 유형 : IaaS, PaaS, SaaS 등
### IaaS

  - Infrastructure as a Service(서비스형 인프라)
  - 서버, 스토리지, 네트워크 등 IT 인프라 자원을 인터넷을 통해 필요한 만큼 빌려 사용하는 서비스
  - 사용자는 물리적인 하드웨어를 직접 구축하고 관리할 필요 없이, 클라우드 제공업체의 인프라를 활용하여 IT 환경을 구축하고 운영할 수 있음
### PaaS

  - Platform as a Service(서비스형 플랫폼)
  - 애플리케이션 개발, 실행, 관리에 필요한 플랫폼을 클라우드 환경에서 제공하는 서비스
  - 활용 사례 : 웹 애플리케이션 개발 및 배포, 모바일 애플리케이션 개발, API 개발 및 관리, 빅데이터 분석 등 
### SaaS

  - Software as a Service(서비스형 소프트웨어)
  - 소프트웨어 애플리케이션을 인터넷을 통해 제공하는 서비스
  - 활용 사례 : 이메일, 협업 도구, CRM, 온라인 오피스 도구 등
### FaaS

  - Function as a Service(서비스형 함수)
  - 서버리스 컴퓨팅의 핵심 기술
  - 개발자는 서버 인프라를 관리할 필요 없이 특정 이벤트에 반응하는 코드(함수)를 작성하고 실행할 수 있음
  - 활용 사례 : API 개발, 실시간 데이터 처리, 이미지.비디오 처리, 챗봇, IoT 백엔드 처리
### XaaS

  - Everything as a Service(서비스형 모든 것)
  - 클라우드 컴퓨팅 환경에서 제공되는 다양한 서비스 모델을 포괄하는 용어
  - IT 자원 뿐만 아니라 다양한 서비스와 솔루션을 인터넷을 통해 제공하는 것
  - 주요 유형 : IaaS, PaaS, SaaS, FaaS, DaaS(Desktop as a Service), DBaaS(Database as a Service), SECaaS(Security as a Service) 등

## 📄 하위 페이지: 클라우드 배포 모델
### 클라우드 배포 모델

  - 조직이 클라우드 컴퓨팅 서비스를 구축, 관리 및 액세스하는 방법을 정의하는 프레임워크
  - 주요 클라우드 배포 모델 : 프라이빗 클라우드, 퍼블릭 클라우드, 멀티 클라우드, 하이브리드 클라우드 등
  - 관련 용어 : 프라이빗/퍼블릭 클라우드, 멀티/하이브리드 클라우드, 커뮤니티 클라우드, 엣지 클라우드 등
### 프라이빗 클라우드

  - 단일 조직에서 독점으로 사용하는 클라우드 환경
  - 조직 내부 데이터센터에 구축하거나 타사 제공업체에서 호스팅 할 수 있음
  - 대표 사례 : OpenStack, VM웨어 등
### 퍼블릭 클라우드

  - 클라우드 서비스 제공업체(CSP)가 인터넷을 통해 일반 대중에게 제공하는 클라우드 컴퓨팅 서비스
  - 대표 사례 : AWS, MS Azure, GCP, 네이버 클라우드, KT 클라우드, NHN 클라우드 등
### 멀티 클라우드

  - 기업이 두 개 이상의 CSP의 클라우드 컴퓨팅 서비스를 사용하는 것
  - 여러 벤더의 퍼블릭 클라우드, 프라이빗 클라우드, 또는 하이브리드 클라우드를 조합하여 사용하는 환경
### 하이브리드 클라우드

  - 퍼블릭 클라우드와 프라이빗 클라우드를 결합하여 사용하는 컴퓨팅 환경
  - 기업이 자체 데이터센터(프라이빗 클라우드)와 외부 클라우드 서비스 제공업체(퍼블릭 클라우드)의 리소스를 함께 활용하는 방식
### 커뮤니티 클라우드

  - 특정 커뮤니티의 여러 조직이 공동으로 사용하는 클라우드 컴퓨팅 환경
  - 공통의 관심사, 요구사항 또는 목표를 가진 조직들이 함께 클라우드 인프라를 공유하고 활용하는 방식
  - 대표 사례 : 정부기관 간, 의료 기관 간, 금융 기관 간, 연구기관 간 커뮤니티 클라우드
### 엣지 클라우드

  - 클라우드 컴퓨팅의 핵심 기능을 데이터센터가 생성되는 물리적 위치 또는 최종 사용자에게 더 가까운 곳에 배치하는 기술
  - 기존의 중앙 집중식 클라우드 컴퓨팅과 달리, 데이터 처리 및 분석을 네트워크의 ‘엣지’에서 수행하여 지연 시간을 줄이고 효율성을 높임
  - 대표 사례 : 자율 주행 자동차, 스마트 공장, 증강 현실(AR) 및 가상 현실(VR), 사물인터넷(IoT) 등

## 📄 하위 페이지: 클라우드 서비스 제공업체
### 클라우드 서비스 제공 업체

  - AWS : 약 32%
  - MS Azure : 약 22%
  - GCP : 약 11%
  - 국내에서는 네이버 클라우드, KT 클라우드, NHN 클라우드 등
### AWS

  - 아마존에서 제공하는 클라우드 컴퓨팅 플랫폼
  - 주요 서비스 : EC2(Elastic Compute Cloud), S3(Simple Storage Service), RDS(Relational Database Service), Lambda, SageMaker 등
### MS Azure

  - 마이크로소프트에서 제공하는 클라우드 컴퓨팅 플랫폼
  - 주요 서비스 : Azure Storage, Azure SQL Database, Azure Functions, Azure AI 등
### GCP

  - 구글에서 제공하는 클라우드 컴퓨팅 서비스
  - 구글 검색, 유튜브, Gmail 등 구글의 서비스에서 사용되는 것과 동일한 인프라를 기반으로 다양한 클라우드 컴퓨팅 서비스 제공
  - 주요 서비스 : Compute Engine, Cloud Storage, BigQuery, Vertex AI 등
  - VM웨어, 레드햇, 델 테크놀로지스, HPE
### VM웨어

  - 가상화 기술의 선두 주자로서, 프라이빗 클라우드 구축에 필수적인 가상화 솔루션 제공
  - 대표 제품군 : vSphere, vSAN, NSX 등
### 레드햇

  - 오픈소스 기반의 프라이빗 클라우드 솔루션 제공
  - Openshift를 통해 컨테이너 기반의 프라이빗 클라우드 환경을 구축할 수 있도록 지원
### 델 테크놀로지스

  - 하드웨어 인프라와 소프트웨어 솔루션을 통합하여 프라이빗 클라우드 구축에 필요한 종합적인 솔루션 제공
  - 대표 제품 : VxRail, PowerEdge 등, 대표 제품군을 통해 HCI 기반의 프라이빗 클라우드 환경을 구축할 수 있도록 지원
### HPE

  - 그린레이크(GreenLake)와 같은 서비스를 통해 프라이빗 클라우드 솔루션을 제공
  - 하드웨어, 인프라, 소프트웨어 및 서비스를 결합하여 기업이 자체 데이터센터에서 클라우드와 유사한 경험을 얻을 수 있도록 지원
### 국내 프라이빗 클라우드 기업

  - 오케스트로, 이노그리드, NHN인재, 에이블클라우드, 나무기술, 효성인포메이션세스템 등
### MSP

  - Managed Service Provider
  - 기업의 IT 시스템 운영 및 관리를 전문적으로 대행하는 서비스를 제공하는 기업

### 클라우드 구현 기술

  - 클라우드 컴퓨팅 환경을 구축하고 운영하는데 필요한 다양한 기술
  - 관련 용어 : IaaS/PaaS를 구현하는 기술, 클라우드 네이티브를 구현하는 기술, CMP, 프로비저닝 등
### IaaS 구현 기술

  - 핵심 요소 : 가상화 기술, 클라우드 관리 플랫폼, 스토리지 플랫폼, 네트워킹 기술, 자동화 및 오케스트레이션 등
  - 관련 용어 : 하이퍼바이저, Ceph, SDN, OpenStack, 자동화 및 오케스트레이션 도구 등
### PaaS 구현 기술

  - 핵심 요소 : 컨테이너 기술, 오케스트레이션 기술, 마이크로서비스 아키텍처, API 관리, 개발 도구 및 서비스 등
  - 관련 용어 : 컨테이너 가상화, 컨테이너 런타임, 컨테이너 오케스트레이션, 쿠버네티스, 클라우드 네이티브 등
### 클라우드 네이티브 구현 기술

  - 클라우드 네이티브 : 클라우드 컴퓨팅 환경의 장점을 최대한 활용하여 애플리케이션을 개발하고 운영하는 방식
  - 클라우드 네이티브 구현 기술 : 클라우드 네이티브를 제대로 구축하기 위해 필요한 핵심 기술
  - 관련 용어 : 컨테이너, 컨테이너 오케스트레이션, DevOps(CI/CD), MSA 등
### 클라우드 통합 운영 및 관리 구현 기술

  - CMP, 자동화 및 오케스트레이션, 모니터링 및 로깅, 비용 관리, 보안 관리, 멀티 클라우드 관리 등
  - 관련 용어 : 프로비저닝, Terraform, Ansible, IaC, ELK Stack, IAM 등

## 📄 하위 페이지: 클라우드 전환
### 클라우드 전환

  - Cloud Migration
  - 기업이나 조직이 기존의 온프레미스(On-Premise) IT 인프라는 클라우드 컴퓨팅 환경으로 이동시키는 과정
### VM 마이그레이션

  - 가상 머신(VM)을 한 호스트 환경에서 다른 호스트 환경으로 이동시키는 프로세스
  - 물리적 서버 간의 VM 이동을 포함하며, 시스템 유지보수, 하드웨어 업그레이드, 데이터센터 이전 등 다양한 상황에 수행
  - 마이그레이션 방식 → 라이브 마이그레이션, 콜드 마이그레이션, 스토리지 마이그레이션
  - 마이그레이션 대상 → P2V, V2V, V2P
### 라이브 마이그레이션

  - VM이 실행 중인 상태에서 중단 없이 다른 호스트로 이동하는 방식
  - 핫 마이그레이션도 동일한 의미
  - 서비스 중단 시간을 최소화하여 고가용성을 유지해야 하는 환경에 적합
### 콜드 마이그레이션

  - VM을 종료한 후 다른 호스트로 이동하는 방식
  - 라이브 마이그레이션에 비해 단순하지만, 서비스 중단 시간이 발생
### 스토리지 마이그레이션

  - VM의 저장소(데이터 스토어)만 다른 호스트로 이동하는 방식
  - 저장 공간 관리 및 성능 최적화에 사용
### P2V

  - Physical to Virtual
  - 물리 서버(Physical Server)에서 설치된 운영체제, 애플리케이션, 데이터를 가상머신(VM)으로 변환하는 마이그레이션 유형
  - 물리적 환경 → 가상환경으로의 전환
### V2V

  - Virual to Virtual
  - 가상머신(VM)을 다른 가상 머신 호스트로 이동시키는 마이그레이션 유형
  - 가상 환경 → 가상 환경으로의 전환 (VM웨어 to OpenStack)
### V2P

  - Virtual to Physical
  - 가상머신(VM)을 물리적 서버(Physical Server)로 변환하는 마이그레이션 유형
  - 가상환경 → 물리적 환경으로서의 전환
### 가트너의 7R 전략

  - 기업이 클라우드로 애플리케이션을 이전할 때 고려해야 할 7가지 전략
  - 각 R은 애플리케이션 마이그레이션 방법
  - 기업은 각 애플리케이션의 특성과 요구 사항에 따라 가장 적합한 전략을 선택
  - Rehost, Replatform, Repurchase, Refactor, Rearchitect, Retain, Retire
